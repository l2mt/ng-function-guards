Guards en Angular

Los Guards en Angular son una forma de controlar y proteger el acceso a rutas específicas en tu aplicación. Al utilizar Guards, puedes asegurarte de que ciertas condiciones se cumplan antes de permitir que el usuario acceda a una ruta o, en algunos casos, antes de abandonar una ruta.

Tipos de Guards

  • CanActivate: Decide si una ruta puede ser activada. Generalmente se utiliza para proteger rutas que requieren autenticación o autorización.
  • CanActivateChild: Decide si una ruta hija puede ser activada. Similar al CanActivate, pero se aplica a las rutas hijas de una ruta en particular.
  • CanDeactivate: Decide si se puede abandonar una ruta. Útil cuando se necesita evitar que el usuario salga de una ruta, como en un formulario con datos no guardados.
  • Resolve: Realiza una acción de resolución de datos antes de activar una ruta. Por ejemplo, puedes obtener datos para una vista antes de que se muestre al usuario.
  • CanLoad: Decide si un módulo de carga diferida (lazy-loaded) puede ser cargado. Útil para evitar la carga de módulos que no son accesibles para un usuario específico.
